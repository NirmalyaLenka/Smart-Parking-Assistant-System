Hardware Stack

What's Inside the System

Every component was chosen for three reasons: it works reliably, it is available everywhere in India, and it keeps the cost under Rs. 300 per parking slot.

COMPONENT 01
Microcontroller Board
ESP8266 NodeMCU v3
The brain of each sensor zone. It reads sensor data, processes it on-board, and transmits results to the server over Wi-Fi using HTTP. The ESP8266 was chosen because it has built-in Wi-Fi — unlike Arduino Uno which needs a separate, expensive Wi-Fi shield. One board can manage up to 8-12 slots with a multiplexer. It runs on 5V USB power, is programmable with the Arduino IDE, and is one of the most documented microcontrollers in the maker ecosystem.
~Rs. 220 per board
COMPONENT 02
Ultrasonic Distance Sensor
HC-SR04
One sensor per parking slot. It emits a 40kHz ultrasonic pulse and measures the time it takes to bounce back from a surface. A vehicle parked in the slot returns the pulse much faster than an empty slot floor — the firmware uses a distance threshold (typically 20-40 cm) to decide if a vehicle is present. The HC-SR04 works from 2 cm to 4 meters, handles cars, bikes, and SUVs without any calibration, and requires only two GPIO pins per sensor.
~Rs. 40 per sensor
COMPONENT 03
Channel Multiplexer
CD74HC4067
The ESP8266 has a limited number of usable GPIO pins. Without a multiplexer, you can only connect about 4-5 HC-SR04 sensors directly. The CD74HC4067 is a 16-channel digital multiplexer that lets one set of ESP8266 pins read up to 16 sensors sequentially. It adds only Rs. 45 to a zone controller but nearly triples the number of slots it can manage, making larger installations practical.
~Rs. 45 per board
COMPONENT 04
Local Backend Server
Raspberry Pi 4 / Any PC
The server runs Node.js and Express. It receives sensor data from all ESP8266 zones over the local Wi-Fi network, stores the current slot state in memory, and serves it to user phones that connect through the QR code. A Raspberry Pi 4 (2GB) costs Rs. 3500 and runs continuously on 5W of power, making it economical for permanent installations. Any old laptop or PC with Linux or Windows works equally well.
~Rs. 3500 (one per site)
COMPONENT 05
QR Code Signage
Printed A5 / A4 Laminated
The QR code is the user-facing entry point. Printed on a laminated sheet and mounted at eye height at the entrance gate, it links to the parking frontend URL. Any phone camera app can scan it. No app download needed. The QR generator script in the repository produces a 1000x1000px PNG suitable for large-format printing, with separate codes for two-wheeler and four-wheeler zones.
~Rs. 20 per sign
COMPONENT 06
Power Supply
5V 2A USB Adapter
Standard USB adapters power the ESP8266 and sensors. An ESP8266 with 6 HC-SR04 sensors draws under 200mA, well within a cheap Rs. 80 adapter. For outdoor zones without a power outlet nearby, a 10W solar panel with a 5V solar charge controller and 3000mAh lithium battery can keep the system running around the clock without grid power.
~Rs. 80 per zone
Why This System

Benefits Over Alternatives

There are several ways to build a parking management system. Here is why this approach was chosen and how it compares to the other common options.

💰
Drastically Lower Cost
Commercial ANPR and RFID parking systems start at Rs. 1.5 lakh for a 20-slot installation. This system covers 20 slots for under Rs. 7000 total, including the server hardware. That is a 95% reduction in upfront cost, with no annual licensing fees.
📱
Zero App Required
RFID card systems require giving every user a card. App-based systems require users to download, register, and log in. This system works with a phone camera and a browser — both of which every smartphone already has. No friction means actual adoption.
⚡
Real-Time Accuracy
Ultrasonic sensors update every 2 seconds. Painted line systems and manual attendants are only as good as the last person who checked. Camera-based AI systems lag by seconds and misfire in rain or darkness. Ultrasonic sensors work in any lighting condition and are unaffected by weather.
🔧
Fully Open Source
Every file is available on GitHub under MIT license. You can modify the firmware for different sensor thresholds, add more zones, change the frontend look, or integrate it into an existing app. There is no vendor lock-in, no call to customer support, and no dependency on a company staying in business.
📏
Separately Tracks Vehicle Types
Two-wheelers and four-wheelers have fundamentally different space requirements. This system tracks them in separate zones with separate counts, so a biker is never directed to a car zone and vice versa. Most budget systems treat all vehicles the same.
💻
Runs on Local Network
The system does not depend on the internet. It works entirely on the local Wi-Fi network at the parking site. This means it continues to function even during internet outages, and there are no cloud hosting costs. Internet connectivity can be optionally added for remote monitoring.

System Architecture

How Data Flows

From the moment a vehicle enters a slot to the moment a driver's phone shows it as occupied — here is every step.

HC-SR04 Sensor → measures distance → ESP8266 NodeMCU → compares threshold → occupied: true/false
ESP8266 NodeMCU → HTTP POST /api/update (Wi-Fi) → Node.js Server → updates in-memory state
Driver's Phone → scans QR code → opens browser → Node.js Server → GET /api/status → Slot Map Displayed
Refresh cycle: ESP8266 posts every 2 seconds — Frontend polls every 3 seconds — All on local Wi-Fi
What Comes Next

Future Scope

The system was designed to be a foundation. These are the natural directions it can grow in, roughly ordered from simplest to most ambitious.

Near Term
Automatic Barrier Control
A servo or relay on the ESP8266 can operate a boom barrier. When four-wheeler slots are all occupied, the barrier stays closed and a display board at the entrance shows a full message. This prevents vehicles from entering and circling a full lot, saving fuel and reducing congestion.
Near Term
LED Indicator Lights Per Slot
A WS2812B addressable LED strip running the length of the parking lot aisle, with one LED per slot, makes it visually obvious where to go without looking at a phone. Green for free, red for occupied. The ESP8266 already has the data — it just needs to drive the LED strip.
Medium Term
UPI Payment Integration
The slot detail view on the user's phone can include a UPI QR code for paying the parking fee by time. The backend can track entry time from the first scan and calculate the fee. No attendant, no cash, no tokens. Integrates with any UPI-enabled payment app already on the user's phone.
Medium Term
Multi-Floor Parking Management
Each floor gets its own set of ESP8266 nodes reporting to the same backend. The frontend adds a floor selector tab. This scales naturally to mall parking structures, hospital parking buildings, and railway station multi-level lots without any architectural change to the core system.
Long Term
License Plate Recognition at Entry
A Raspberry Pi camera at the entrance gate running OpenALPR or a custom YOLO model can log which vehicle entered, when, and which slot it parked in. This creates a complete audit trail useful for security and dispute resolution, without requiring drivers to present anything.
Long Term
Smart City Integration
Municipal corporations can deploy this system across hundreds of public and street-side parking locations and feed real-time availability into a city-wide parking portal or navigation app. Research has consistently shown that 30-40% of urban traffic congestion is caused by drivers looking for parking. Live city-wide parking data directly addresses this problem at scale.
Long Term
Solar-Powered Off-Grid Deployment
A 10W solar panel, a small battery pack, and a low-power Wi-Fi mesh network can make entire parking lots grid-independent. This opens deployment in rural markets, pilgrimage sites, agricultural mandis, and event grounds where grid electricity is unreliable or expensive to bring in.
